# BGGデータ同期自動化作戦っ！ (๑•̀ㅂ•́)و✧

ボドゲのミカタのデータ同期を自動化して、BGGとのラグをゼロにするためのインフラ構築戦略だよ！

## 1. 自動化の基本方針 (๑˃̵ᴗ˂̵)و
手動でのトリガーなんて古いもん！インフラ側でガッチリ同期するよっ！

*   **完全独立型バックグラウンドプロセス**: FastAPIアプリとは別に、BGG同期専用のCLIツールを動かすの！
*   **Playwright活用**: ネットワークリクエストは全部Playwrightで処理して、BGGにやさしく同期するよ。
*   **増分更新**: 全部取ってくるんじゃなくて、必要な分だけ更新するから効率的でしょ？

## 2. 実装計画 (❁´◡`❁)

### Step 1: 同期スクリプトの開発
`scripts/bgg_sync.py`を作成して、`bgg_service`を呼び出してSQLiteを最新にするよっ！

### Step 2: 定期実行のインフラ設定
`Taskfile.yml`に同期タスクを追加して、`systemd`ユーザーサービスで一定間隔（例えば5分ごと）で実行するの！
```yaml
tasks:
  sync:bgg:
    cmds:
      - uv run python -m scripts.bgg_sync
```

### Step 3: 監視とエラーハンドリング
同期に失敗したら、ログに残して管理者に知らせるようにするから安心だよ！(๑•̀ω•́๑)

## 3. なんでこの方法なの？ (•'-'•)
*   **0%ラグ**: 定期実行を短くすることでラグを最小限にできるもん！
*   **インフラ独立**: FastAPIのパフォーマンスに影響を与えないから快適っ！
*   **堅牢性**: Playwrightなら動的なサイト構成変更にも強いから、安心だよっ！

この作戦で、ボドゲのミカタを最強のボドゲデータエンジンにしちゃおう！応援よろしくねっ！！
