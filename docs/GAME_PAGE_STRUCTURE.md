# ボドゲのミカタv2: ゲームページ設計 (コンテキスト・ファースト)

## 1. デザイン哲学: "Data as Servant, Context as King"
BGGの「詳細データの羅列」は、もはやユーザーを困らせるだけです。私たちは「そのゲームを遊ぶまでの心理的・物理的ハードルをどう下げるか」に全振りします。

## 2. 構造のレイヤー設計

### 1. 感情の羅針盤 (Emotional Compass) - Hook
*   **キャッチコピー:** どんな気分の時に出すゲーム？
*   **体験の核心:** プレイ時間、複雑さよりも「何をやりたいか」を3単語で表現。
*   **インタラクション強度:** 協力系か、殴り合い系か、ソロパズル系かを直感的なビジュアルで表示。

### 2. プレイの真実 (The Reality of Play) - Context
*   **ティーチング難易度:** 5分で教えられるか？それとも30分必要か？
*   **準備の摩擦:** セットアップに何分かかるか？
*   **プレイヤー数スケール:** 「何人で遊ぶのが最適か？」をユーザーの声で補完。

### 3. 洞察の集積 (Synthesis Hub) - Utility
*   **NotebookLMによる戦略のショートカット:** 複雑な戦術を「最初の3手」に圧縮。
*   **共通の落とし穴:** 誰でも一度はやる失敗リスト。
*   **FAQ (ボドゲ特化):** ルールのグレーゾーンを明快に。

### 4. 基礎データ (Raw Stats) - Data Foundation
*   **必要最小限:** プレイ時間、人数、重量級(Weight)、メカニズム(厳選して3つまで)。
